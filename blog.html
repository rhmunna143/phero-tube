Discuss the scope of var, let, and const

var:

  • Functional level or global scope.
  • Raised and initialized with undefined.
  • It can be reassigned and re-declared.

let:
  • Block-level scope.
  • Raised but uninitialized (if used before declaration results in ReferenceError).
  • It can be reallocated but cannot be declared again in the same scope.

const:
  • Block-level scope.
  • Raised but uninitialized (if used before declaration results in ReferenceError).
  • Cannot be reassigned after declaration (but object properties or array elements can be modified).
  • Cannot be declared again in the same scope.

Tell us the use cases of null and undefined

null:

  • It is used to explicitly express the absence of a value.
  • It is often used as a placeholder for future assignments.

undefined:
  • Automatically assigned to function parameters that are not available.
  • Represents the default value for uninitialized variables.
  • It is usually used to indicate that something is missing or undefined.
  • It can be unintentional and indicate a bug in your code when you encounter it unexpectedly.

What do you mean by REST API?

REST API, or Representational State Transfer Application Programming Interface, is a widely used web services architecture. It prioritizes simplicity and scalability, allowing software components to communicate effectively. Key principles include stateless status where every application contains all necessary information; client-server model for separation of interests; uniform interfaces with standardized HTTP methods such as GET and POST; and resource-based URLs. These features simplify the development of web applications and enable the exchange of data between clients and servers. REST APIs have become an essential part of the development of modern web and mobile applications and support seamless data acquisition and manipulation.
